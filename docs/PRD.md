# 产品需求文档：个人财务助手（FinFlow）

---

## 1. 产品概述

### 1.1 产品定位

> **FinFlow** 是一款面向个人的智能财务分析与管理工具。它不同于传统的记账软件（仅记录流水）或仪表盘（仅展示图表），而是通过**数据驱动 + 规则引擎**，主动帮助用户理解财务状况、发现潜在风险、制定并追踪财务目标。

### 1.2 产品愿景

让每个普通人都能拥有一个**24 小时在线的私人财务分析师**——不只看钱花在哪，更帮你规划钱的去向。

### 1.3 核心差异化

| 维度 | 传统记账软件 | FinFlow |
|------|-------------|---------|
| 数据输入 | 手动记账，依赖用户持续录入 | 导入银行流水 + 自动清洗分类，降低录入成本 |
| 分析深度 | 分类汇总（本月餐饮花了多少） | 多维度诊断（季节性模式、结构性变化、异常检测） |
| 交互方式 | 用户主动查看报表 | 主动推送预警、建议、月度报告 |
| 规划能力 | 无 | 目标管理 + 反向计算 + 情景推演 |
| 数据闭环 | 只记录历史 | 历史→诊断→预测→规划→追踪 全闭环 |

## 2. 目标用户

### 2.1 用户画像

| 属性 | 描述 |
|------|------|
| 年龄 | 25-40 岁 |
| 收入 | 月薪 10k-50k，有稳定工资和奖金 |
| 财务复杂度 | 有储蓄卡/信用卡/基金/股票/公积金/外债中至少 3 种 |
| 痛点 | 知道要理财，但不清楚自己真实财务状况；月底钱花完了对不上账；想存钱但不知道从哪省 |
| 技术能力 | 会用手机 App，能操作网页，**不应要求会改代码** |

### 2.2 用户故事（优先级排序）

```
US-01  作为用户，我希望导入银行流水后系统自动分类，不需要我手动标注每一笔
US-02  作为用户，我希望打开首页就能清楚知道 "我现在有多少钱、能花多少钱、钱在哪"
US-03  作为用户，我希望系统主动提醒我预算超支了，而不是等我去翻报表
US-04  作为用户，我希望知道 "我的钱主要花在哪些地方，哪些是必要的、哪些可以省"
US-05  作为用户，我想设定一个目标（比如 3 年存 30 万），系统告诉我每月该存多少、现在进度如何
US-06  作为用户，我想看到我的资产是变多了还是变少了，趋势如何
US-07  作为用户，我希望系统告诉我 "你的财务状况健康吗"，并给出具体的改进建议
US-08  作为用户，我想看我的持仓到底赚了还是亏了，哪些该卖哪些该留
US-09  作为用户，我想知道自己如果换工作/买车/买房，财务状况会怎么变化
US-10  作为用户，我希望每年自动收到一份财务年报，总结全年的收支和资产变化
```

## 7. 数据模型（核心实体）

```
User             用户（本地版暂为单用户）
├─ AssetAccount    资产账户（招行卡、微信零钱...）
│  ├─ layer        层级（现金/投资/受限/应收）
│  └─ snapshots[]  月度快照
│
├─ Transaction     交易记录（导入的流水）
│  ├─ category_l1  一级分类
│  ├─ category_l2  二级分类
│  ├─ is_outlier   异常标记
│  └─ tags[]       用户标签
│
├─ Budget          预算配置
│  └─ items[]      {分类, 预算金额, 周期(月/年)}
│
├─ Goal            财务目标
│  ├─ type         类型（攒钱/资产/支出控制）
│  ├─ target       目标值
│  ├─ deadline     目标日期
│  └─ progress[]   进度记录
│
├─ Alert           预警记录
│  ├─ type         类型（预算/流动性/异常）
│  ├─ severity     严重程度
│  ├─ message      消息
│  └─ read_at      已读时间
│
└─ Report          财务报告
   ├─ type         月度/年度
   └─ generated_at 生成时间
```

## 8. 附录

### 8.1 术语表

| 术语 | 定义 |
|------|------|
| 可用资金 | 现金/活期类资产总和，即时可支配 |
| 流动性覆盖率 | 可用资金 ÷ 月均支出，单位：月 |
| 投资占比 | 投资资产 ÷ 总资产 × 100% |
| 结构性变化 | 生活重大事件导致的支出模式改变（买车、换房） |
| 财务健康评分 | 综合流动性/储蓄率/收益率/负债率/分散度的加权评分 |

### 8.2 竞品参考

| 产品 | 优势 | 不足 |
|------|------|------|
| 随手记/挖财 | 记账体验好，用户基数大 | 缺乏深度分析，广告多 |
| 鲨鱼记账 | 快速记账，UI 简洁 | 不做规划，不做预测 |
| 记账城市 | 游戏化，有趣 | 分析功能弱 |
| MoneyWiz / YNAB | 预算管理成熟，多平台 | 国外产品，中文支持差 |
| **FinFlow** | **深度分析 + 预测规划 + 本地隐私** | **记账体验弱于竞品** |

### 8.3 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 用户不愿导入历史数据 | 冷启动困难 | 提供预设模板和示例数据 |
| 预测模型不准 | 用户不信任 | 显示置信区间 + 回测准确率 |
| 数据安全顾虑 | 用户不敢用 | 本地存储、不开源账户、可脱敏导出 |
| 移动端缺失 | 低频使用 | 优先保证响应式 Web 可用 |
